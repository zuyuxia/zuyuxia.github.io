C++语法基础知识

发表于 2018-03-15 | 分类于 C++

语法知识点

  1. union

    • union是一种特殊的类，一个union可以有多个数据成员，但是任意时刻只有一个数据成员有值

    • 分配个一个union对象的存储空间至少能容纳它最大的数据成员

  2. const的作用

    • 定义const变量，具有不可变性

    • 进行类型检查，是编译器对处理内容有更多的了解，消除隐患

    • 方便参数的调整和修改

    • 保护被修饰的东西，防止被意外修改

    • 提供函数的重载

    • 节省空间，避免不必要的内存分配。相比#define，const定义的常量在程序中只有一份复制品

  3. ASSERT与assert

    • ASSERT是宏，只能用在Debug版本，Release版本被忽略；assert是函数，Debug和Release版本都可以使用

    • 两者都是用于判断程序中是否出现了非法数据

  4. float类型判断为是否为0

    • float和double类型的变量在内存中的存储机制与整型数不同，有舍入误差。大多数浮点数是无法精确表达的

    • 判断相等时，使用范围来确定：x >= low && x <= high

  5. C++中异常的处理方法和关键字

    • 异常处理的关键字：try、catch、throw

    • C++中的异常处理机制只能处理由throw捕获的异常，没有捕获的将被忽略

    • 使用try{}catch(){}语句来捕获异常，把可能发生异常的代码放到try{}语句块中，后面跟上若干个catch(){}负责处理具体的异常类型

  6. 回调函数

    • 回调函数就是被调用者回头调用的函数，通过函数指针调用。如果把函数的指针作为参数传递给另一个函数，当使用指针调用其指向的函数时，就可称其为回调函数

    • 回调函数不是由该函数的实现方直接调用，而是在特定的事件或条件发生时由另一方调用，来对事件或条件进行相应

    • 定义和实现一个回调函数：声明、定义、设置触发条件

  7. 判断大小端 

    union un{
    int i;  
    char ch;  
};
void fun(){
    union un test;
    test.i = 1;
    if(ch == 1)
        cout << "小端" << endl;
    else
        cout << "大端" << endl;
}

    • 大端：高位字节放在内存的低地址端，低位字节放在内存的高地址端
      小端：低位字节放在内存的低地址端，高位字节放在内存的高地址端

    • 大端：符号位的判定固定为第一个字节，容易判断正负
      小端：强制转换数据不需要调整字节内容

  8. ++i 不是原子操作

    • ++i 分为三个步骤：
      • 将数据从内存放在寄存器上
      • 在寄存器上自增
      • 把数据从寄存器上拷贝回内存

  9. const在成员函数中的位置以及相应的意义

    • const修饰返回值，如fun1函数，意味着fun1函数返回值是一个const int

    • const放在参数列表之后，被称为常量成员函数

      • const的作用在于修改隐式this指针的类型；默认情况下this的类型是指向类类型非常量版本的常量指针，即Test *const

      • 常量成员函数不能改变调用它的对象的内容

      • 常量成员函数只能调用常量成员函数；而非常量成员函数能够调用常量成员函数

      • 只有常量成员函数才能被常量对象调用

      class Test{
public:
   const int fun1();
   int fun2() const;
};

  10. 变量声明和定义的关系

    • 变量声明和定义都规定了变量的类型和名字；但定义还申请了内存空间，声明并不申请存储空间

    • 变量只能被定义一次，但可以被多次声明

  11. sizeof和strlen的区别

    • sizeof是一个运算符，结果在编译时得到；strlen是一个函数，结果在运行中得到

    • sizeof作用于任何类型或者数据（即使是空指针或者野指针）；strlen函数的参数只能是字符指针（const char *str）

    • strlen(“\0”) = 0, sizeof(“\0”) = 2

    • 补充：sizeof由于在编译时就会获得结果，所以不能用来获取动态分配的空间大小；sizeof返回值是一个常量表达式

  12. 关于static

    • static可以修饰局部变量、全局变量和函数，被修饰的变量存储在静态区；静态局部变量只在第一次被调用时定义，生命期直到程序运行结束，但作用域只在局部作用域；静态全局变量只能在本文件中可见，相对于全局变量，可见范围变小；修饰函数时，同全局变量一样，限定了访问域。

    • 修饰类成员：静态成员变量和静态成员函数；为所有对象共用；不能在类声明中定义，静态成员变量的初始化在类外；静态成员函数不包含this指针，不能被声明成const，无法访问对象的非静态成员变量和非静态成员函数

  13. 内存分配的形式

    • 由符号其实的区块段（BSS）：通常用来存放程序中未初始化的全局数据和静态数据；BSS段属于静态内存分配，程序结束后静态变量资源由系统自动释放

    • 数据段（data segment）：通常用来存放程序中已初始化的全局变量；数据段也属于静态内存分配

    • 代码段（code segment）：存放程序执行代码；大小在程序运行前就已经确定

    • 堆（heap）：存放进程运行中被动态分配的内存段，大小不固定，可动态扩张、缩小；new/malloc

    • 栈（stack）：存放程序临时创建的局部变量

  14. （C++）struct和class有什么区别

    • 默认限定符：struct是public，class是private

  15. 类中的成员函数不能用const和static修饰

    • C++编译器在实现const成员函数时为了保证函数不会修改类中参数的值，会在函数中添加一个隐式的参数const this*。static修饰成员函数时，函数时没有this指针的。因此，两者会冲突

  16. malloc和new的区别

    • 概念：new和delete是运算符，支持重载；malloc和free是函数（cstdlib），支持覆盖

    • malloc在堆上分配内存块，new申请的内存在自由存储区上

    • malloc和free仅仅分配和回收空间，不调用构造函数和析构函数；new和delete，则会调用构造函数和析构函数

    • new返回的是带有具体类型的指针；malloc返回的是void指针，需要进行类型转换

  17. 栈空间的大小

    • 栈是向低地址扩展的，是一块连续的内存区域。栈顶地址和栈的最大容量是系统预先规定好的

    • Linux默认栈空间大小8MB，windows为2MB

  18. 引用和指针的区别

    • 指针本身是一个对象，允许对指针赋值和拷贝；引用本身不是一个对象，所以不能定义引用的引用

    • 指针可以先后指向多个不同的对象；引用只能作为一个变量的引用，无法绑定到另一个对象

    • 指针声明和定义可以分开；引用在声明时必须定义，绑定初始值

    • 不存在指向空值的引用，必须有具体实体；存在指向空值的指针

  19. 宏定义和函数的区别

    • 宏在编译之前完成替换，替换的代码参与编译；函数调用在运行时需要跳转到具体的函数代码

    • 宏属于直接替换代码，不具有返回值；函数调用可以具有返回值

    • 宏的参数没有类型，不进行类型检查；函数的参数具有类型，需要检查类型

    • 格式问题：宏最后不需要加分号

  20. extern关键字

    • 设置与变量或函数前，表示变量或者函数的定义在别的文件中

    • extern还可以用来进行链接指定：extern “C”；C++在编译的时候会将函数名和参数联合起来生成一个中间的函数名称，而C语言则不会，因此会造成链接时找不到对应函数的情况，此时C函数就需要用extern “C”进行链接指定，告诉编译器，不要生成用于链接的中间函数名。

  21. volatile的作用

    • 当对象的值可能在程序的控制或检测之外被改变时，应当将该对象声明为volatile，系统总是重新从所在内存中读取数据

    • volatile告诉编译器不应当对这样的对象进行优化

    • 一个参数既可以是const也可以是volatile：只读状态寄存器

    • 使用场景：程序debug版本和release版本结果不一样

  22. 为什么要内存对齐

    • 平台原因：考虑到不同硬件访问数据的兼容性

    • 性能原因：CPU访问内存的速度大大提升

  23. 常量引用（常引用）

    • 概念：把引用绑定到const对象上

    • 常量引用下，不能用作修改它所绑定的对象

  24. 指针常量和常量指针

    • 指针常量（const pointer），必须初始化，指向的地址不能变化，地址的值能够变化

    • 常量指针（pointer to const），指向一个只读变量

  25. 野指针

    • 概念：空悬指针，不是指向nullptr的空指针，指向垃圾内存（随机内存）

    • 产生原因：未及时初始化；free或delete后没有及时置为nullptr

  26. 智能指针如何实现，什么时候该改变

    • 多个智能指针shared_ptr可以指向同一个对象，一旦最后一个这样的指针被销毁，这个对象会被自动删除

    • 构造函数中计数初始化为1；拷贝构造函数中计数值+1；赋值运算符中，左边的对象引用计数-1，右边的+1；析构函数中引用计数-1；

  27. 堆和栈的区别

    • 申请方式：栈由系统自动分配；堆由程序员手动分配

    • 申请大小限制：栈顶和栈底都是已经预设好的，大小固定；堆向高地址扩展，是不连续的内存区域，大小可以灵活调整

    • 申请效率不同：栈分配速度快，不会有碎片；堆分配速度慢，有碎片

  28. C++四种类型转换：static_cast、dynamic_cast、const_cast、reinterpret_cast

    • const_cast：用于将const变量转换为非const

    • static_cast：任何具有明确定义的类型转换，只要不包含底层const，都可以使用static_cast

    • dynamic_cast：用于动态类型转换，只能用于含有虚函数的类，只能转指针或引用

    • reinterpret_cast：几乎什么都可以转换，但是很有可能出现问题，尽量少用

  29. 必须在构造函数初始化式里进行初始化的数据成员

    • 常量成员

    • 引用类型

    • 没有默认构造函数的类

  30. #include<filename.h>和#include “filename.h”的区别

    • <filename.h>，编译器先从标准库路径开始搜索filename.h，系统文件调用较快

    • “filename.h”，编译器先从用户工作路径开始搜索filename.h，再去寻找系统路径，自定义文件较快

  31. 最有效的计算2^n的方法

    • 直接进行乘法运算也可以计算，但是并不是最优方法，通过移位运算更加高效

    • 快速求某个数的7倍：先将数左移3位（扩大八倍），再减去原值。即通过移位和加减运算来代替乘除运算

  32. 位操作求两个数的平均值

    • 一般来说，求平均数的方法：(x + y)/2

    • x,y很大时上面的方法会溢出。可以采用位运算：(x & y) + ((x ^ y) >> 1)
