C++语法基础知识

发表于 2018-03-15 | 分类于 C++

#语法知识点

  1. const在成员函数中的位置以及相应的意义

    • const修饰返回值，如fun1函数，意味着fun1函数返回值是一个const int

    • const放在参数列表之后，被称为常量成员函数

      • const的作用在于修改隐式this指针的类型；默认情况下this的类型是指向类类型非常量版本的常量指针，即Test *const

      • 常量成员函数不能改变调用它的对象的内容

      • 常量成员函数只能调用常量成员函数；而非常量成员函数能够调用常量成员函数

      • 只有常量成员函数才能被常量对象调用

  2. 变量声明和定义的关系

    • 变量声明和定义都规定了变量的类型和名字；但定义还申请了内存空间，声明并不申请存储空间

    • 变量只能被定义一次，但可以被多次声明

  3. sizeof和strlen的区别

    • sizeof是一个运算符，结果在编译时得到；strlen是一个函数，结果在运行中得到

    • sizeof作用于任何类型或者数据（即使是空指针或者野指针）；strlen函数的参数只能是字符指针（const char *str）

    • 补充：sizeof由于在编译时就会获得结果，所以不能用来获取动态分配的空间大小；sizeof返回值是一个常量表达式

  4. 关于static

    • static可以修饰局部变量、全局变量和函数，被修饰的变量存储在静态区；静态局部变量只在第一次被调用时定义，生命期直到程序运行结束，但作用域只在局部作用域；静态全局变量只能在本文件中可见，相对于全局变量，可见范围变小；修饰函数时，同全局变量一样，限定了访问域。

    • 修饰类成员：静态成员变量和静态成员函数；为所有对象共用；不能在类声明中定义；静态成员函数不包含this指针，不能被声明成const，无法访问对象的非静态成员变量和非静态成员函数

  5. （C++）struct和class有什么区别

    • 默认限定符：struct是public，class是private

  6. malloc和new的区别

    • 概念：new和delete是运算符，支持重载；malloc和free是函数（cstdlib），支持覆盖

    • malloc在堆上分配内存块，new申请的内存在自由存储区上

    • malloc和free仅仅分配和回收空间，不调用构造函数和析构函数；new和delete，则会调用构造函数和析构函数

    • new返回的是带有具体类型的指针；malloc返回的是void指针，需要进行类型转换

    • 关于malloc和new的一些参考博客

      • 细说new与malloc的10点区别
      • C++ 自由存储区是否等价于堆

  7. 引用和指针的区别

    • 指针本身是一个对象，允许对指针赋值和拷贝；引用本身不是一个对象，所以不能定义引用的引用

    • 指针可以先后指向多个不同的对象；引用只能作为一个变量的引用，无法绑定到另一个对象

    • 指针声明和定义可以分开；引用在声明时必须定义，绑定初始值

    • 不存在指向空值的引用，必须有具体实体；存在指向空值的指针

  8. 宏定义和函数的区别

    • 宏在编译之前完成替换，替换的代码参与编译；函数调用在运行时需要跳转到具体的函数代码

    • 宏属于直接替换代码，不具有返回值；函数调用可以具有返回值

    • 宏的参数没有类型，不进行类型检查；函数的参数具有类型，需要检查类型

    • 格式问题：宏最后不需要加分号

  9. extern关键字

    • 设置与变量或函数前，表示变量或者函数的定义在别的文件中

    • extern还可以用来进行链接指定：extern “C”；C++在编译的时候会将函数名和参数联合起来生成一个中间的函数名称，而C语言则不会，因此会造成链接时找不到对应函数的情况，此时C函数就需要用extern “C”进行链接指定，告诉编译器，不要生成用于链接的中间函数名。

    • 参考资料：

      • C/C++中extern关键字详解

  10. volatile的作用

    • 当对象的值可能在程序的控制或检测之外被改变时，应当将该对象声明为volatile，系统总是重新从所在内存中读取数据

    • volatile告诉编译器不应当对这样的对象进行优化

    • 一个参数既可以是const也可以是volatile：只读状态寄存器

    • 参考资料：

      • C/C++中volatile关键字详解

  11. 为什么要内存对齐

    • 平台原因：考虑到不同硬件访问数据的兼容性

    • 性能原因：CPU访问内存的速度大大提升

  12. 常量引用（常引用）

    • 概念：把引用绑定到const对象上

    • 常量引用下，不能用作修改它所绑定的对象

  13. 指针常量和常量指针

    • 指针常量（const pointer），必须初始化，指向的地址不能变化，地址的值能够变化

    • 常量指针（pointer to const），指向一个只读变量

  14. 野指针

    • 概念：空悬指针，不是指向nullptr的空指针，指向垃圾内存（随机内存）

    • 产生原因：未及时初始化；free或delete后没有及时置为nullptr

  15. 智能指针如何实现，什么时候该改变

    • 多个智能指针shared_ptr可以指向同一个对象，一旦最后一个这样的指针被销毁，这个对象会被自动删除

    • 构造函数中计数初始化为1；拷贝构造函数中计数值+1；赋值运算符中，左边的对象引用计数-1，右边的+1；析构函数中引用计数-1；

  16. 堆和栈的区别

    • 申请方式：栈由系统自动分配；堆由程序员手动分配

    • 申请大小限制：栈顶和栈底都是已经预设好的，大小固定；堆向高地址扩展，是不连续的内存区域，大小可以灵活调整

    • 申请效率不同：栈分配速度快，不会有碎片；堆分配速度慢，有碎片

  17. C++四种类型转换：static_cast、dynamic_cast、const_cast、reinterpret_cast

    • const_cast：用于将const变量转换为非const

    • static_cast：任何具有明确定义的类型转换，只要不包含底层const，都可以使用static_cast

    • dynamic_cast：用于动态类型转换，只能用于含有虚函数的类，只能转指针或引用

    • reinterpret_cast：几乎什么都可以转换，但是很有可能出现问题，尽量少用

  18. 必须在构造函数初始化式里进行初始化的数据成员

    • 常量成员

    • 引用类型

    • 没有默认构造函数的类
