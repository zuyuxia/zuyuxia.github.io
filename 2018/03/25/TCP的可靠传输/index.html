TCP的可靠传输

发表于 2018-03-25 | 分类于 计算机网络

注：文中图片均来源于谢希仁《计算机网络》第六版的课件PPT

TCP的可靠传输

以客户端(client)向服务器(server)发送数据为例，TCP采用以下措施来保证可靠传输

  • 确认和超时重传：服务器在接收到客户端发来的数据之后都要给出确认；而客户端在发送数据之后，就会设置一个定时器，超过这个时间还没收到相应的确认，就要重传数据

  • 数据校验：在TCP的头部存在一个校验和字段，用来检验数据是否出错

  • 分片和排序：客户端TCP会根据MTU对数据进行合理的分片，并编号；服务器会缓存未按序到达的数据，重新排序好后再上交给应用层

  • 流量控制：在服务器来不及处理客户端发送来的数据时，通过滑动窗口机制让客户端发送数据的速度降低

  • 拥塞控制：在探寻到网络状况较差时，减少数据发送，避免网络状况进一步恶化

确认和超时重传

  1. 客户端为A，服务器为B，如下图所示

    • A在向B发送了41号数据，并且收到了33号数据的确认；B则按序接收到了26~33号数据，还接收了部分的未按序到达数据，等待到缺失字节到达后再上交给应用层

    • 累积确认的好处在于不用发送那么的确认号，坏处在于可能需要重传已经接收了的数据

  2. 超时重传
    在一定时间内没有收到相应报文的确认，会进行重传数据；对于这个时间的设定，十分复杂，这里介绍一下《计算机网络》书中的方法：

    • TCP采用一种自适应的算法，记录一个报文段的发出时间，以及收到相应确认的时间，两者之差就是报文段的往返时间RTT。TCP保留了RTT的一个加权平均往返时间RTTS。
    • RTTS的计算：第一次测量到RTT样本时，RTTS值就取为所测量到的RTT样本值，之后的RTTS按照下式进行计算：（a的推荐值为0.125） 
      新的RTTS=(1-a)\*旧的RTTS + a\*新的RTT样本
    • 超时重传时间RTO按照下式计算： 
      RTO=RTTS + 4*RTTD
    • RTT的偏差加权平均值RTTD的计算：第一次测量时，取为RTT样本值的一半，以后的测量中，按下式进行计算： 
      新的RTTD=(1-b)\*旧的RTTD + b\*| RTTS - 新的RTT样本 |
    • Karn算法：在计算加权平均RTTS时，只要报文段重传了，就不采用其往返时间样本。但是这种算法会导致超时重传时间无法更新
    • 修正的Karn算法：报文每重传一次，就把超时重传时间RTO增大一些，典型做法是取为旧的重传时间的2倍。当不再发送报文重传时，再根据上述的公式进行计算RTTS和RTO

  3. 选择确认SACK
    未按序到达的数据一般都被接收方存下来了，如果能够只传输缺少的数据不重传已经接收的数据可以提高效率，选择确认SACK就是这种方法

    • 选择确认SACK工作原理：TCP的接收方在接收对方发过来的数据字节流的序号不连续，形成一些不连续的字节快，前后字节不连续的每一个字节块都有两个边界。使用SACK能够描述这些字节块的边界信息。

    • 如果需要使用SACK，那么在建立TCP时，就要在首部的选项中加入允许SACK的选项，并且双方必须事先商定好。由于选项的长度最多只有40字节，指明一个边界需要4字节，最多只能指明4个字节块的边界信息。
