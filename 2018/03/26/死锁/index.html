死锁

死锁的定义：如果一组进程中的每一个进程都在等待仅由该组进程中的其它进程才能引发的事件，那么该组进程就是死锁的

产生死锁的必要条件

  1. 互斥条件

    • 进程对所分配到的资源进行排他性使用，在一段时间内，资源只能被一个进程使用，如果有其他进程请求该资源，则请求进程进入等待状态，直到资源被释放

  2. 请求和保持条件

    • 进程已经获得了一个资源，但是又申请了一个新的资源，该资源被另一个进程占用，于是进程进入等待状态，但是进程并不释放已经获得的资源

  3. 不可抢占条件

    • 进程已获得的资源在未使用完之前不能被其他进程抢占，除非进程自己释放资源

  4. 循环等待条件

    • 发生死锁时，存在一个进程-资源的循环链。{P0，P1，P2……PN}，P0等待P1，P1等待P2……PN等待P0

  5. 处理死锁的办法

    • 预防死锁：通过设置某些限制条件，破坏产生死锁的四个必要条件中的一个或者多个来预防产生死锁

    • 避免死锁：在资源的动态分配过程中，用某种方法防止系统进入不安全的状态，从而避免产生死锁

    • 检测死锁：无需事先采取措施，允许进程进入死锁状态，通过检测机构检测出死锁的发生，采取适当的措施，解除死锁状态

    • 解除死锁：与检测死锁想配合，检测出死锁状态后采取相应措施解除死锁状态。常用的方法是撤销一些进程，回收资源，分配资源给已经阻塞的进程

避免死锁：银行家算法

  1. 银行家算法思路

    • 银行家算法就是在进程获得资源之前确认系统是否有足够的资源分配给进程，若有足够的资源就接纳进程，此外还要看分配资源给进程之后是否会导致死锁。如果会死锁那么就不分配资源

    • 进程可以分次请求资源，但是请求总量不能超过最大需求量

    • 当系统现有的资源不能满足进程所需资源时，可以推迟分配，但是总应该在一定的时间内为进程分配到所需资源，避免饥饿现象

  2. 银行家算法中的数据结构

    • 可利用资源Available：Available[i] = K，表示系统中现有K个i资源

    • 最大需求矩阵Max：Max[i,j] = K，表示进程i需要K个j资源

    • 分配矩阵Allocation：Allocation[i,j] = K，表示进程i已经分到K个j资源

    • 需求矩阵Need：Need[i,j] = K，表示进程I还需要K个j资源才能完成任务

  3. 银行家算法
    Request是进程i请求的资源量，假设Request[j] = K，在进程i发出请求后，系统按照以下步骤进行检查

    • 如果Request[j] <= Need[i,j]，进入下一个步骤；否则认为出错

    • 如果Request[j] <= Available[i,j]，进入下一步骤；否则系统没有那么多j资源，进程i必须等待

    • 系统尝试将资源j分配给进程i，并修改上面的四个数据结构的数值

    • 系统执行安全性算法，检测此次分配资源后，系统是否仍处于安全状态。若安全则分配，否则，恢复资源分配状态，进程i进入等待状态

  4. 安全性算法

    • 设置两个变量：工作变量Work，表示系统可提供给进程继续运行所需各种资源的数目，在安全算法的开始，Work = Available；Finish，表示系统是否有足够的资源完成任务，开始时Finish[i] = false，当有足够的资源分配给进程时，再置为true

    • 第一步，从进程中找到一个满足下面条件的进程，找到执行第二步，否则进入第三部

      • Finish[i] = false;
      • Need[i,j] <= Work[j];

    • 第二步，当进程i得到资源后，顺利执行，释放所占用的资源，返回第一步

      • Work[j] = Work[j] + Allocation[i,j];
      • Finish[i] = true;

    • 第三步，如果所有进程的Finish[i] = true，则系统处于安全状态，否则不安全
