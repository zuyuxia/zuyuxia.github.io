DNS：域名系统

发表于 2018-03-28 | 分类于 计算机网络

DNS：域名系统

  • 主机的一种标识方法就是使用主机名，主机名也就是我们通常所说的域名。虽然主机名便于记忆，但是对于机器而言，主机名不是定长，也没有任何提示主机在网络中的位置。因此对于主机，也可用使用IP地址标识出主机的位置。于是，我们需要一种域名和IP地址之间的转换服务—DNS域名系统

  • DNS：一个由分层的DNS服务器实现的分布式的数据库，一个使得主机能够查询分布式数据库的应用层协议

  • DNS协议运行在UDP之上，使用53号端口

DNS提供的服务

  • 当用户浏览器请求一个URL时发送的事件：

    • 主机上运行着DNS应用的客户端

    • 浏览器从URL中得到主机名，并将主机名传给DNS应用的客户端

    • DNS客户向DNS服务器发送一个包含主机名的请求

    • DNS客户会收到一份应答报文，其中包含对应于主机名的IP地址

    • 浏览器在得到IP地址后，便可向IP地址所在的服务器发起一个TCP连接

  • DNS提供的其他服务：主机别名，邮件服务器别名，负载分配

DNS工作机理概述

DNS服务器的类型：根DNS服务器、顶级域DNS服务器、权威DNS服务器、本地DNS服务器

  1. 以主机A想知道主机B的IP地址为例：

    • 主机A先向本地DNS服务器发送DNS查询报文，若有记录则直接返回，否则执行下一步骤
    • 本地DNS服务器将报文转发到根DNS服务器，根服务器向本地DNS服务器返回相应的TLD服务器的IP地址列表
    • 本地DNS服务器再次向这些TLD服务器之一发送查询报文，TLD将响应的权威服务器IP地址发送给本地DNS服务器
    • 最后，本地服务器直接向权威服务器发送查询报文
    • 主机到本地DNS服务器是递归查询，其余的都是迭代查询

  2. DNS缓存

    • DNS缓存目的：减少网络中的查询流量
    • 原理：在一个请求链中，当某DNS服务器接收到一个DNS回答时，将回答信息缓存在本地存储器中
    • 本地DNS服务器能够缓存TLD服务器的IP地址，允许本地DNS绕过查询链中的根DNS服务器
    • DNS服务器在一段时间（两天）将丢弃缓存信息
