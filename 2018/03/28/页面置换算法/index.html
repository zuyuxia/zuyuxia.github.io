页面置换算法

发表于 2018-03-28 | 分类于 操作系统

注：文中图片均来源于汤子瀛《计算机操作系统》第四版

页面置换算法

在进程运行过程中，其要访问的页面不在内存之中，那么就需要将页面调入内存，如果此时内存已满，则需要将内存中的某一页调到交换区。这种确定将哪一页调出的算法称为页面置换算法。

页面置换算法如果不合适会造成进程的抖动。所谓的进程抖动，就是刚刚被换出的页面很快又会被访问，而下一次被换出的页面也会很快被访问，如此频繁的更换页面，使得进程在运行过程中把大部分时间浪费在页面置换上。

最佳置换算法（OPT）

  • 最佳置换算法是一种理想化的算法，具有最好的性能，但是由于无法预测未来，所以无法实现。通常用来作为标准，衡量其他算法的优劣

  • OPT每次发生缺页中断时，选择被淘汰的页将是以后不使用或者很长时间内不会被使用的页面

先进先出置换算法（FIFO）

  • 先进先出置换算法是最直观的算法，实现简单，但通常与页面的使用规律不符合，性能较差

  • FIFO每次在发生缺页中断时，选择最先进入内存的页面进行淘汰，即先进入内存，先退出内存。实现起来较为容易，采用队列即可实现

最近最久未使用置换算法（LRU）

  • 最近最久未使用置换算法是根据页面调入内存后的使用情况作出决策的，由于无法预测页面未来的使用情况，采用最近的过去来作为最近的将来的近似

  • LRU置换算法是选择最近最久未使用的页面淘汰，该算法赋予每一个页面一个访问字段，用来记录该页面从上次被访问到目前所经历的时间t。当发生缺页中断时，选择t值最大的页面予以淘汰

  • LRU算法是向前看，根据各页以前的使用情况来判断；最佳置换算法是向后看，依据各页以后的使用情况判断

  • LRU算法的硬件支持：寄存器和栈

    • 寄存器：为了记录某进程在内存中各页的使用情况，必须为每个在内存中的页面配置一个移位寄存器；进程访问某物理块时，相应的寄存器要置为1，定时信号定时将寄存器右移一位；如果把n位寄存器的数看作是一个整数，那么具有最小数值的寄存器所对应的页面就是最近最久未使用的页面

    • 栈：利用一个特殊的栈保存当前使用的各个页面的页面号。每当进程访问某页面时，将页面号从栈中移出，压入栈顶；因此栈顶始终是最新的被访问的页面的编号，栈底则是最近最久未使用页面的页面号

最少使用置换算法（LFU）

  • 采用LFU算法时，应该在内存中为每一个页面设置一个移位寄存器，用来记录该页面被访问的频率

  • LFU选择在最近时期使用最少的页面作为淘汰页。每次访问某页时，将该页的移位寄存器最高位置为1，然后再定期向右移位。在发送缺页中断时，寄存器每一位上1的和最小的页将被换出

Clock置换算法

LRU算法是一种比较好的算法，但是需要较多的硬件支持，实现起来成本比较高，在实际应用中，大多采用LRU的近似算法。Clock算法就是用的较多的一种LRU近似算法

  • 简单的Clock置换算法：为每一页设置一个访问位，然后将页面通过链接指针链接成一个循环队列。当某一页被访问时，访问位就置为1。在发生缺页中断时，检查访问位，若为0，就换出；若为1，则置为0，暂不换出，将指针移到下一页。该算法又被称为最近未用算法（NRU）

  • 改进型Clock置换算法：在简单Clock算法基础上，增加一个修改位，即不仅考虑页面使用情况，还考虑置换代价。选择页面时，既要是未使用过的页面，又要是未被修改过的页面

    • 页面可分为四种类型：1类（A=0，M=0）;2类（A=0，M=1）;3类（A=1，M=0）;4类（A=1，M=1）

    • 缺页中断时的置换过程：第一轮扫描，寻找1类页面；如果第一轮扫描未找到1类页面，执行第二轮扫描，寻找2类页面，在扫描中将每一页的访问位A置为0；若第二轮扫描也失败，重复寻找1类页面，重复寻找2类页面，则一定能找到要被淘汰的页
