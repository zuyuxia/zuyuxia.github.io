1、开机流程

  • 开机 -> BIOS(读取第一个扇区的MBR位置) -> MBR(引导加载程序，加载内核文件) -> 内核文件（开始操作系统的功能）

2、文件可存取访问的三个身份：owner、group、others

3、权限：read、write、execute

4、linux用户身份与用户组记录的文件

  • /etc/passwd：所有系统上的账号
  • /etc/shadow：个人的密码
  • /etc/group：所有的组名

5、ls -al：显示的内容含义：

  • (文件类型与权限)(有多少个不同文件连接到相同的i-node)(所有者)(所属用户组)(容量大小)(最近修改日期)(文件名)

6、文件的类型与权限：(0)(123)(456)(789)，一共十个字符，数字表示位置索引

  • 重要性：数据安全，数据共享
  • 0：表示文件类型
    • d：目录
    • -：文件
    • l：连接文件
    • b：可供存储的接口设备
    • c：串行端口设备
  • 123：表示文件所有者的权限（rwx），没有权限则为-
  • 456：表示同用户组权限
  • 789：表示非本用户组的其他者权限

7、目录权限中的x表示是否能进入该目录

8、linux文件种类

  • 普通文件：纯文本文件，二进制文件，数据文件（cat打开会乱码，用last打开）
  • 目录
  • 连接文件
  • 设备与设备文件：块设备文件，字符设备文件
  • 套接字
  • 管道

9、特殊的目录：

  • .：当前目录
  • ..：上一层目录
  • ~：当前用户的主文件夹
  • -：前一个工作目录
  • ~user：user用户的主文件夹

10、文件的三个时间：

  • mtime：modification time，文件内容进行更改时，就会更新这个时间
  • ctime：status time，文件的状态进行更改时，更新这个时间，例如权限和属性
  • atime：access time，当文件内容被取用时，更新这个时间

11、文件默认权限

  • 文件的默认权限：666
  • 目录的默认权限：777
  • 当前文件、目录的权限=默认权限-umask分数

12、whereis，locate，find之间的区别

  • 三个命名都是查找文件
  • whereis和locate是利用数据库来查询，linux系统会将系统内的所有文件记录在一个数据库文件中
  • find是直接查找硬盘
  • locate是从数据库/var/lib/mlocate/里面的数据进行查找，更新数据库的操作为：updatedb

13、windows文件系统：NTFS，Linux：Ext2/Ext3，U盘：FAT

  • Linux文件系统（索引式文件系统）通常分为三部分：superblock、inode、block
    • superblock：记录文件系统的整体信息，包括inode/block的总量，使用量，剩余量以及文件系统格式等信息
    • inode：记录文件属性，一个文件占用一个inode，同时记录该文件数据所在的block号码
    • block：实际记录文件的内容，文件太大时会占用多个block
  • U盘没有inode，只有block，每个block号码记录在前一个block中
    • 碎片整理：需要碎片整理的原因在于记录文件内容的block过于分散，文件读取的性能比较差，需要将这些block会和在一起，提高性能
