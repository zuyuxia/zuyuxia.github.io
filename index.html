网络模型和网络设备

发表于 2018-04-12 | 分类于 计算机网络

网络模型

  1. OSI七层模型

    • 应用层：主要负责确定通信对象，并确保有足够的资源用于通信。常见的应用层协议有HTTP、FTP、SNMP

    • 表示层：负责数据的编码以及转化，同时负责进行数据的压缩、解压，加密和解密，确保应用层能够正常工作

    • 会话层：主要负责在网络中的两个结点之间建立、维护、控制会话，区分不同的会话，提供单工、半双工、全双工三种通信模式

    • 传输层：负责分割、组合数据，实现端到端的逻辑连接。TCP、UDP、SPX

    • 网络层：将网络地址翻译为物理地址，决定将数据从发送方路由到接收方。IP、RIP、OSPF

    • 数据链路层：控制物理层与网络层之间的通信，主要负责物理传输的准备

    • 物理层：规定激活、维持、关闭通信端点之间的机械特性、电气特性、功能特性和过程特性

发表于 2018-04-11 | 分类于 操作系统

Linux内存管理

  1. Linux内存管理：用户空间和内核空间

    • 以4G进程地址空间为例，用户空间为0-3G，内核空间为3-4G；通常用户只能访问用户空间，进行系统调用时才能访问内核空间

    • 用户空间对应进程，进程切换时用户空间也变化；内核空间由内核负责映射，并不会随进程变化而变化

    • 用户空间：数据段，代码段，BSS端，堆，栈；内核空间：ZONE_DMA，ZONE_NORMAL，ZONE_HIGHMEM

  2. 高端内存（ZONE_HIGHMEM）

    • 高端内存值物理地址大于896M的内存，对于这样的内存，无法在“内核直接映射空间”进行映射；引入高端内存的主要原因在于内核的1G线性地址空间无法建立一个完全的直接映射来访问整个物理内存空间

    • ZONE_HIGHMEM：896M~结束

    • 内核能够借助高端内存访问所有物理内存：当内核想要访问物理地址高于896M的内存时，会从高端内存区借一段地址空间，建立临时地址映射，使用完毕后释放

静态链接（库）和动态链接（库）

发表于 2018-04-11 | 分类于 操作系统

静态链接和动态链接

  • 静态链接是指把要调用的函数或者过程直接链接到可执行文件中，成为可执行文件的一部分。即函数和过程的代码就在程序的exe文件之中，该exe文件包含了运行时所需的全部代码。静态编译的缺点时当多个运行程序都调用同一个函数时，内存中会存在这个函数的多个拷贝，浪费资源

  • 动态链接是相对于静态链接而言的，动态链接所调用的函数代码并没有被链接到可执行文件中，只是在其中加入了调用函数的一些描述信息。当程序装入内存运行时，在操作系统的管理下，才在应用程序与相应的动态链接库之间建立链接关系。在执行动态链接库中的函数代码时，根据链接产生的重定位信息，转去执行动态链接库中的函数代码

  • 静态链接的可执行文件可在同类操作系统的其他主机上运行，而动态链接的可执行文件则必须包括相应的动态链接库文件

静态链接库和动态链接库

  • 静态链接库中的代码最后需要链接到可执行文件之中

  • 动态链接库是一个包含可由多个程序同时使用的代码和数据的库，包含函数和数据的模块的集合

  • windows下：静态库.lib；动态库.dll

  • Linux下：静态库.a；动态库.so

内存池、进程池和线程池

发表于 2018-04-11 | 分类于 操作系统

内存池

默认的new/delete、malloc/free在堆上分配和释放内存，由于频繁的调用和每一次申请的内存块大小不定，会造成大量的内存碎片并且降低性能

内存池是一种内存的分配方式。在使用真正内存之前，先分配一块大的连续的内存块作为备用（一般是大小相等，数量一定）。当有内存需求时，从内存池中分出一部分内存块给其使用，内存使用完毕后，并不归还给系统，而是归还给内存池。这减少了new/delete、malloc/free的调用次数，提高了内存分配效率

内存池可以分为单线程内存池和多线程内存池

内核态和用户态

内核态和用户态是操作系统的两种运行级别，区分不同程序的不同权限。如果一个程序运行在内核态，则可以访问计算机的任何资源，访问权限最高。如果一个程序运行在用户态，那么其资源访问权限受到各种限制。

  • 内核态下CPU可执行任何执行；用户态下CPU只能执行非特权指令

  • CPU处于内核态时，可随意进入用户态；CPU处于用户态时，只能通过系统调用和中断切换到内核态

  • 运行在内核态的程序权限高，可访问资源多，可靠性和安全性要求高；用户态的程序权限低，可访问资源少，但是可靠性和安全性的要求低

  • 内核态：CPU管理、内存管理、诊断和测试程序、输入输出管理以及文件系统的部分管理；用户态：文件系统的用户数据管理、编辑器用户程序和编译器、网络管理的部分功能

哲学家进餐问题

  • 问题的描述：有五个哲学家围着一张餐桌，分别坐在五张椅子上，在桌子上有五根筷子。他们的活动方式为交替思考和用餐，即哲学家一直思考，饥饿的时候就试图取两边的筷子进行用餐，只有拿到两根筷子时才能用餐。用餐完毕后，放下筷子继续思考。

  • 这是一个经典的同步问题，若五个人同时拿起左（右）边的筷子，那么就会形成死锁

  • 三种解决办法：

    • 至多允许有四位哲学家拿左边的筷子

    • 仅当哲学家两边的筷子均可用时，允许用餐

    • 规定奇数号的哲学家先拿左边筷子，偶数号哲学家先拿右边筷子

进程同步

为了能够保证多个进程能够有条不紊的进行作业，在多道程序系统中，必须引入进程同步机制。进程同步机制是对多个相关进程在执行次序上进行协调，使并发的进程之间能够按照一定的规则共享系统资源，能够相互合作，程序的执行具有可再现性。单处理机系统中的进程同步机制有：硬件同步机制、信号量机制、管程机制
