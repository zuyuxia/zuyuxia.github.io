面向对象

发表于 2018-03-16 | 分类于 C++

#面向对象知识

  1. 面向对象的三大特性

    • 封装：数据和代码捆绑在一起，避免外界干扰和不确定性的访问

    • 继承：让某种类型对象获得另一种类型对象的属性和方法

    • 多态：同一事物表现出不同事物的能力，即向不同对象发送同一消息，不同的对象在接收时产生不同的行为；重载实现编译时多态，虚函数实现运行时多态

  2. public/private/protected

    • public：类内，类外

    • private：类内

    • protected：类内，派生类

  3. 对象的存储空间

    • 非静态成员的数据类型大小之和

    • 为了边缘对其加入的panding

    • 比一期加入的额外成员变量（指向虚函数的指针）

    • 静态数据、虚函数不占对象的存储空间

    • 参考博客：

      • C++中类对象的内存布局和占用空间

  4. 虚基类

    • 通过虚继承解决基类在多条链路上被一个子类继承引发的二义性问题；虚继承的目的是让某个类共享它的基类，共享的基类子对象称为虚基类

    • 在子类的构造函数列表中，必须显示的调用虚基类的构造函数

    • 构造函数执行顺序：虚基类早于非虚基类

  5. 纯虚函数和抽象基类

    • 纯虚函数为派生类提供了一个一致的接口：virtual void fun() = 0;

      • 一般的时候，一个纯虚函数是不需要定义的，由派生类来实现

      • 可以实现纯虚函数，提供一种缺省行为，给派生类调用

    • 抽象基类：含有纯虚函数的类是抽象基类

      • 抽象基类不能实例化，可以定义指向抽象基类的指针和引用

      • 抽象基类中既能有抽象方法，也能有非抽象方法；继承于抽象基类的子类，只有实现了所有父类的抽象方法才是非抽象类

    • 参考博客：C++ 虚函数&纯虚函数&抽象类&接口&虚基类

  6. 接口与多继承

    • 接口：类型提供的操作，不需要完成特定类的实现，通常情况下，接口不包含数据成员

    • C++中通过抽象基类实现接口；当继承一个接口时，接口里的所有函数必须全部被覆盖

    • 多继承：从多个直接基类中产生派生类

    • 多继承比单一继承复杂，可能会导致新的歧义性；多继承也有优点，public继承接口类，private继承某些实现

  7. C++空类

    • 空类的大小为1个字节

    • 默认函数：构造函数、析构函数、拷贝构造函数、赋值运算符（一个类没定义这些拷贝控制成员，编译器会自动为它定义缺失的操作）

  8. 构造函数或者析构函数能否为虚函数

    • 构造函数不能定义为虚函数，且构造函数中还不能调用虚函数（语法上没有问题，但是往往不能达到需要的目的，此外构造顺序：先基类后派生类）

    • 析构函数能定义为虚函数，并且一般情况下基类析构函数要定义为虚函数；只有基类析构函数为虚函数时，调用delete销毁指向对象的基类指针时，才能准确调用派生类的析构函数准确销毁数据；析构函数可以是纯虚函数，

    • 参考博客：

      • 为什么基类的析构函数是虚函数

  9. 构造函数和析构函数的调用顺序问题

    • 构造函数：（first）纵向上有多个基类先调用最上层的基类构造函数，横向上有多个基类调用顺序按照派生表从左到右顺序；（second）类变量中包含其他类，即成员类对象的构造函数，按照在类中被声明的先后顺序调用相应构造函数；（third）调用本类构造函数

    • 析构函数的调用顺序与构造函数的相反

  10. 拷贝构造函数中的深拷贝和浅拷贝的区别

    • 深拷贝时，当被拷贝对象存在动态分配的内存空间时，需要先动态申请一块内存空间，然后逐字节拷贝内容

    • 浅拷贝仅仅拷贝指针字面值，使用浅拷贝时，如果原来的对象被释放掉了数据，那么会产生空悬指针，即野指针

  11. 拷贝构造函数和赋值运算符重载的区别

    • 拷贝构造函数是函数，赋值运算符是运算符重载

    • 拷贝构造函数会生成新的对象，而赋值运算符只能在已经生成了的对象之间作用，不会生成新的对象

    • 拷贝构造函数是直接构造一个新的类对象，初始化对象前不需要检查源对象和新建对象是否相同；赋值运算符检查，并且提供两套不同的复制策略，赋值运算符中如果原来的对象有内存分配还需要把内存释放掉

    • 形参传递是调用拷贝构造函数，并不是所有出现”=”的地方都是使用赋值运算符

  12. 虚函数和纯虚函数的区别

    • 虚函数是为了实现动态绑定，目的是通过基类指针指向不同对象的时候，自动调用派生类中和基类同名的函数。虚函数需要在普通函数前加上修饰符virtual，virtual会被隐式继承，所以子类中同名的函数都是虚函数，且函数名、返回值类型、参数类型和个数都要与基类函数的相同

    • 纯虚函数，仅仅相当于是一个接口名，含有纯虚函数的类不能够实例化

  13. 重载，覆盖和隐藏

    • 重载是相同函数名但参数个数、类型或顺序都可以不一样的两个函数，调用函数时，根据传递的参数列表来确定调用哪个函数。可以使类外函数也可以是类内函数

    • 覆盖是派生类中重新定义的函数，除函数体外其他都与基类函数相同，覆盖只能发生在类内函数

    • 隐藏是指派生类中函数屏蔽了与其同名的函数

  14. 在main执行之前的代码可能是什么

    • 全局对象的构造函数

    • 参考博客：

      • Main函数之前和之后执行代码

  15. 虚指针

    • 虚指针或虚函数指针是实现虚函数的细节

    • 虚指针指向虚表结构

  16. 虚函数的实现问题

  1. 重载与函数模板

    • 重载只需要函数名相同，参数列表、个数等无所谓

    • 函数模板则参数个数相同，参数类型无所谓

  2. this指针

    • this指针是类的指针，指向对象的首地址；只能在成员函数中使用，静态函数和全局函数中都不能使用this指针

    • this指针的存储位置因编译器而异

  3. 类模板

    • 为解决多个功能相同而类型不同的类之间需要重复定义的问题

    • 对一批仅数据成员类型不同的类的抽象

  4. 构造函数和析构函数调用时机

    • 全局范围中的对象：构造函数在所有函数调用之前执行，在主函数执行完调用析构函数

    • 局部自动对象：建立对象时调用构造函数，函数结束时调用析构函数

    • 动态分配的对象：建立对象时调用构造函数，调用释放时调用析构函数

    • 静态局部变量对象：建立时调用一次构造函数，主函数结束时调用析构函数

  5. C++多态的实现

    • 多态分为静态多态和动态多态

    • 静态多态：通过重载和模板技术实现，编译时确定

    • 动态多态：通过虚函数和继承关系实现，执行动态绑定，运行时确定

    • 动态实现的几个条件：

      • 虚函数
      • 一个基类的指针/引用指向派生类的对象

C++语法基础知识

发表于 2018-03-15 | 分类于 C++

#语法知识点

  1. const在成员函数中的位置以及相应的意义

    • const修饰返回值，如fun1函数，意味着fun1函数返回值是一个const int

    • const放在参数列表之后，被称为常量成员函数

      • const的作用在于修改隐式this指针的类型；默认情况下this的类型是指向类类型非常量版本的常量指针，即Test *const

      • 常量成员函数不能改变调用它的对象的内容

      • 常量成员函数只能调用常量成员函数；而非常量成员函数能够调用常量成员函数

      • 只有常量成员函数才能被常量对象调用

  2. 变量声明和定义的关系

    • 变量声明和定义都规定了变量的类型和名字；但定义还申请了内存空间，声明并不申请存储空间

    • 变量只能被定义一次，但可以被多次声明

  3. sizeof和strlen的区别

    • sizeof是一个运算符，结果在编译时得到；strlen是一个函数，结果在运行中得到

    • sizeof作用于任何类型或者数据（即使是空指针或者野指针）；strlen函数的参数只能是字符指针（const char *str）

    • 补充：sizeof由于在编译时就会获得结果，所以不能用来获取动态分配的空间大小；sizeof返回值是一个常量表达式

  4. 关于static

    • static可以修饰局部变量、全局变量和函数，被修饰的变量存储在静态区；静态局部变量只在第一次被调用时定义，生命期直到程序运行结束，但作用域只在局部作用域；静态全局变量只能在本文件中可见，相对于全局变量，可见范围变小；修饰函数时，同全局变量一样，限定了访问域。

    • 修饰类成员：静态成员变量和静态成员函数；为所有对象共用；不能在类声明中定义；静态成员函数不包含this指针，不能被声明成const，无法访问对象的非静态成员变量和非静态成员函数

  5. （C++）struct和class有什么区别

    • 默认限定符：struct是public，class是private

  6. malloc和new的区别

    • 概念：new和delete是运算符，支持重载；malloc和free是函数（cstdlib），支持覆盖

    • malloc在堆上分配内存块，new申请的内存在自由存储区上

    • malloc和free仅仅分配和回收空间，不调用构造函数和析构函数；new和delete，则会调用构造函数和析构函数

    • new返回的是带有具体类型的指针；malloc返回的是void指针，需要进行类型转换

    • 关于malloc和new的一些参考博客

      • 细说new与malloc的10点区别
      • C++ 自由存储区是否等价于堆

  7. 引用和指针的区别

    • 指针本身是一个对象，允许对指针赋值和拷贝；引用本身不是一个对象，所以不能定义引用的引用

    • 指针可以先后指向多个不同的对象；引用只能作为一个变量的引用，无法绑定到另一个对象

    • 指针声明和定义可以分开；引用在声明时必须定义，绑定初始值

    • 不存在指向空值的引用，必须有具体实体；存在指向空值的指针

  8. 宏定义和函数的区别

    • 宏在编译之前完成替换，替换的代码参与编译；函数调用在运行时需要跳转到具体的函数代码

    • 宏属于直接替换代码，不具有返回值；函数调用可以具有返回值

    • 宏的参数没有类型，不进行类型检查；函数的参数具有类型，需要检查类型

    • 格式问题：宏最后不需要加分号

  9. extern关键字

    • 设置与变量或函数前，表示变量或者函数的定义在别的文件中

    • extern还可以用来进行链接指定：extern “C”；C++在编译的时候会将函数名和参数联合起来生成一个中间的函数名称，而C语言则不会，因此会造成链接时找不到对应函数的情况，此时C函数就需要用extern “C”进行链接指定，告诉编译器，不要生成用于链接的中间函数名。

    • 参考资料：

      • C/C++中extern关键字详解

  10. volatile的作用

    • 当对象的值可能在程序的控制或检测之外被改变时，应当将该对象声明为volatile，系统总是重新从所在内存中读取数据

    • volatile告诉编译器不应当对这样的对象进行优化

    • 一个参数既可以是const也可以是volatile：只读状态寄存器

    • 参考资料：

      • C/C++中volatile关键字详解

  11. 为什么要内存对齐

    • 平台原因：考虑到不同硬件访问数据的兼容性

    • 性能原因：CPU访问内存的速度大大提升

  12. 常量引用（常引用）

    • 概念：把引用绑定到const对象上

    • 常量引用下，不能用作修改它所绑定的对象

  13. 指针常量和常量指针

    • 指针常量（const pointer），必须初始化，指向的地址不能变化，地址的值能够变化

    • 常量指针（pointer to const），指向一个只读变量

  14. 野指针

    • 概念：空悬指针，不是指向nullptr的空指针，指向垃圾内存（随机内存）

    • 产生原因：未及时初始化；free或delete后没有及时置为nullptr

  15. 智能指针如何实现，什么时候该改变

    • 多个智能指针shared_ptr可以指向同一个对象，一旦最后一个这样的指针被销毁，这个对象会被自动删除

    • 构造函数中计数初始化为1；拷贝构造函数中计数值+1；赋值运算符中，左边的对象引用计数-1，右边的+1；析构函数中引用计数-1；

  16. 堆和栈的区别

    • 申请方式：栈由系统自动分配；堆由程序员手动分配

    • 申请大小限制：栈顶和栈底都是已经预设好的，大小固定；堆向高地址扩展，是不连续的内存区域，大小可以灵活调整

    • 申请效率不同：栈分配速度快，不会有碎片；堆分配速度慢，有碎片

  17. C++四种类型转换：static_cast、dynamic_cast、const_cast、reinterpret_cast

    • const_cast：用于将const变量转换为非const

    • static_cast：任何具有明确定义的类型转换，只要不包含底层const，都可以使用static_cast

    • dynamic_cast：用于动态类型转换，只能用于含有虚函数的类，只能转指针或引用

    • reinterpret_cast：几乎什么都可以转换，但是很有可能出现问题，尽量少用

  18. 必须在构造函数初始化式里进行初始化的数据成员

    • 常量成员

    • 引用类型

    • 没有默认构造函数的类
