页面置换算法

在进程运行过程中，其要访问的页面不在内存之中，那么就需要将页面调入内存，如果此时内存已满，则需要将内存中的某一页调到交换区。这种确定将哪一页调出的算法称为页面置换算法。

页面置换算法如果不合适会造成进程的抖动。所谓的进程抖动，就是刚刚被换出的页面很快又会被访问，而下一次被换出的页面也会很快被访问，如此频繁的更换页面，使得进程在运行过程中把大部分时间浪费在页面置换上。

单例模式

  • 单例模式又称为单子模式，是一种常见的设计模式。在使用这个模式时，单例对象的类必须只有一个实例存在。

  • C++中单例模式分为懒汉和饿汉实现：懒汉意味着第一次用到类实例的时候才会实例化，饿汉指单例类在定义的时候就进行了实例化

  • 线程同步，访问量比较大的时候，采用饿汉实现可以实现更好的性能；访问量较小时，采用懒汉实现

操作系统内存管理：页式、段式、段页式

连续分配的方式会形成许多碎片，虽然可以通过紧凑的方式将许多碎片拼接成可用的大块空间，但需要很大的开销。如果能够将一个进程分散的装入到许多不相邻的分区中，那么可以充分利用内存空间，也不需要采用紧凑的方法去拼接碎片。基于这种离散分配的思想，根据分配地址空间的基本单位不同的，可以将离散分配分为：分页、分段和段页式

TCP的可靠传输

发表于 2018-03-25 | 分类于 计算机网络

注：文中图片均来源于谢希仁《计算机网络》第六版的课件PPT

TCP的可靠传输

以客户端(client)向服务器(server)发送数据为例，TCP采用以下措施来保证可靠传输

  • 确认和超时重传：服务器在接收到客户端发来的数据之后都要给出确认；而客户端在发送数据之后，就会设置一个定时器，超过这个时间还没收到相应的确认，就要重传数据

  • 数据校验：在TCP的头部存在一个校验和字段，用来检验数据是否出错

  • 分片和排序：客户端TCP会根据MTU对数据进行合理的分片，并编号；服务器会缓存未按序到达的数据，重新排序好后再上交给应用层

  • 流量控制：在服务器来不及处理客户端发送来的数据时，通过滑动窗口机制让客户端发送数据的速度降低

  • 拥塞控制：在探寻到网络状况较差时，减少数据发送，避免网络状况进一步恶化

红黑树

红黑树是一种平衡二叉排序树，在插入和删除操作时通过相应的操作保持平衡，从而能够获得较高的查找性能，插入、删除和查找的时间复杂度是O(logN)

  1. 红黑树的性质
    • 树中的结点要么是红色，要么是黑色
    • 根结点是黑色
    • 每个叶节点（NULL结点）是黑色的
    • 每个红色结点的两个子结点都是黑色，不能有两个连续的红色结点
    • 任一结点到其叶子结点的路径中包含的黑色结点数目相同

虚拟内存

发表于 2018-03-20 | 分类于 操作系统

注：文中图片均来源于汤子瀛《计算机操作系统》第四版

虚拟内存

  • 虚拟内存是计算机系统内存管理的一种技术。它使得应用程序认为它拥有连续可用的内存（一个连续完整的地址空间），而实际上，它通常是被分隔成多个物理内存碎片，还有部分暂时存储在外部磁盘存储器上，在需要时进行数据交换。与没有使用虚拟内存技术的系统相比，使用这种技术的系统使得大型程序的编写变得更容易，对真正的物理内存（例如RAM）的使用也更有效率
    引自维基百科：虚拟内存

  • 作用：

    • 无需在作业运行时一次性地装入内存，允许分成多次调入内存运行

    • 无需在作业运行时一直常驻内存

    • 从逻辑上扩充内存的容量，是用户看到的内存容量

  • 虚拟内存的实现：

    • 请求分页存储管理

    • 请求分段存储管理

    • 请求段页式管理

    • 所需的硬件支持：一定容量的内存和外存；页表机制或段表机制；中断机构；地址变化机构
